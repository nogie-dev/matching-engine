# Matching 처리 영역 메모

**범위/역할**
- 한 오더북(티커) 단위로 매칭한다. 서로 다른 티커 간 교차 매칭은 없다.
- 매처는 “교차 여부 판단 → 체결 계산 → 체결 이벤트 생성”만 담당하는 함수/컴포넌트로 최소화한다.
- 호출은 함수형이 적합(`Match(book, order)`), 워커가 소유한 오더북을 넘기고 잔여만 `AddOrder`에 적재한다.
- 라우터는 티커별 워커만 관리하며 이벤트를 워커 채널로 전달한다.

**트리거/흐름**
- 이벤트(New/Cancel/Edit) 처리 직후 워커 고루틴 안에서 바로 매칭을 호출한다(bestBid >= bestAsk 해소될 때까지). 취소/감소도 단순화를 위해 한 번 돌릴 수 있다.
- 새 주문: 반대편 교차 매칭을 먼저 돌리고 잔여가 있을 때만 `AddOrder`(교차 없으면 바로 적재).
- 수정(Edit): 가격을 더 공격적으로 바꾸거나 수량을 늘린 경우 수정 반영 후 매칭을 돌린다. 단순화를 위해 모든 수정 후 매칭 한 번 실행도 가능.

**매칭 루프 세부**
- 비교 조건: Bid는 `incoming.Price >= bestAsk.Price`, Ask는 `incoming.Price <= bestBid.Price`, 루프에 `incoming.Amount > 0` 포함.
- 베스트 호가를 힙 `Peek`으로 보고, 해당 가격대 큐 맨 앞을 `Front`로 확인해 부분/전량 체결한다. 전량일 때만 `Remove`.
- 레벨이 비면 힙/맵에서 레벨 삭제(`dropPriceLevel`).
- 잔여 처리: 매칭 후 `incoming.Amount <= 0`이면 nil 반환, 잔여가 있으면 그대로 반환해 책에 적재한다.
